What is "Remember: A Link to the Past"?

The Remember: A Link to the Past is an open-source solution developed for companies and teams looking to optimize information management and retrieval. It periodically captures the user's screen, processes the images with artificial intelligence, and stores the data in a database. With an intuitive interface, it allows for quick and efficient searches of any past moments.

The advanced technology of Remember centralizes and organizes all essential information, ensuring quick and accurate access to documents, data, and insights, transforming the team's workflow.

Key Benefits:

  • Intelligent Search : Quickly recover documents and records with the help of AI.
  • Centralized Organization : Store and access information securely and conveniently.
  • Real-Time Collaboration : Agile sharing and integration with other tools.
  • Privacy and Security : Full control over data with local storage options.
  • Productivity : Increase efficiency with fast access to information.
  • Cost-Effective : Robust solution without the need for advanced hardware.

Who is it Ideal for:

  • Companies of all sizes that need to optimize data organization and retrieval.
  • IT professionals and managers looking to centralize team knowledge.
  • Marketing, sales, and operations teams that need agility to access information.

Workflow and Features

  1. Automatic Screenshot Capture : Captures screenshots at defined time intervals and stores them locally.
  2. AI Image Processing : Captured images are processed to identify open applications, transcribe texts, and generate descriptions and other metadata.
  3. Vectorized Database Storage : Extracted descriptions and metadata are saved and vectorized in a database, allowing fast and efficient searches in the future.
  4. Search for Past Events : Through a simple interface, users can search previous moments using an efficient semantic search to find specific events.

1. Screenshot Capture

To capture computer screens every minute. 

import pyautogui
import time

def captura_tela(intervalo=60, caminho='./screenshots/'):
    while True:
        screenshot = pyautogui.screenshot()
        screenshot.save(f"{caminho}{time.time()}.png")
        time.sleep(intervalo)

The screenshot function can be optimized in various ways to generate a timestamp automatically and store images in an organized manner, supporting multiple directories based on the date.

3. Storage and Search

Processed data is stored and can be searched later for relevant content. 

from vector_store import armazenar_dados, buscar_por_conteudo

def salvar_e_pesquisar(descricao, conteudo_busca):
    armazenar_dados(descricao)
    return buscar_por_conteudo(conteudo_busca)

For a simplified approach, we can use Ollama Embeddings to generate document vectors and FAISS to perform similarity searches.

The code can easily be optimized to run locally and connect with a server configured to handle custom embeddings. 

embeddings = OllamaEmbeddings(base_url=ollama_server_url, model='nomic-embed-text')
vectorstore = FAISS.from_documents(documents, embeddings)

4. Search Interface

Implementing a simple interface to facilitate search. 

def interface_de_busca():
    busca = input("Enter what you want to find: ")
    resultado = salvar_e_pesquisar(busca)
    print(f"Found result: {resultado}")

With an interface built using Streamlit , users can easily search their activity history using a search bar. The interface presents relevant results, including image content, extracted text, and a link to the capture location in the history.

Advanced Technical Data

The project uses accessible technologies with great potential for expansion and customization:

  • Screenshot Capture : Implemented with the PyAutoGUI library, configurable for regular intervals.
  • Image Processing : The AI processes the captures using models such as PHY3 , OpenAI CLIP or equivalents.
  • Data Storage : Descriptions are saved in relational or NoSQL databases like SQLite , MongoDB or PostgreSQL .
  • Vector Search : Information is vectorized in a vector store , allowing fast and efficient searches through embeddings.
